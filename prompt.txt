Here's the full breakdown of what was found and fixed across all 15 issues:

🔴 Binary Pipeline — Critical Bugs Found
1. SVM probability=False (your "ROC-AUC: Not available" bug) — The SVM lacked probability=True, so predict_proba() threw an exception and the entire ROC/threshold analysis was silently skipped. Fixed.
2. XGBoost scale_pos_weight=None — This is why XGBoost showed Non-Disclosure recall of only 0.31 despite 0.81 accuracy. The model almost never predicted class 0. For binary XGBoost, scale_pos_weight = neg_count / pos_count ≈ 0.35. Fixed by computing it from y_train at runtime.
3. Missing class_weight on LR, RF, SVM — None of your ML models had class weighting. RF now uses balanced_subsample (per-tree balancing, better than global balanced).
4. BERT missing warmup scheduler + gradient clipping — This directly explains the 465 False Positives vs 3 False Negatives — the model became overconfident/aggressive at predicting positive because large early gradients destabilized pre-trained weights. Fixed with get_linear_schedule_with_warmup and clip_grad_norm_(1.0).
5. BiLSTM threshold hardcoded at 0.7 in phase6_error_analysis.py — Error analysis and evaluation use different thresholds, making FP/FN counts inconsistent. Fixed by passing threshold from evaluate().

🟢 Multiclass Pipeline — Critical Bugs Found
6. Missing sklearn imports in evaluator.py — classification_report, confusion_matrix, etc. are all used but never imported. The multiclass evaluator would crash immediately at runtime.
7. Zero threshold analysis for multiclass — Binary had Youden/F1/G-Mean optimization. Multiclass had nothing. Implemented full One-vs-Rest per-class threshold optimization and a apply_multiclass_thresholds() function that falls back to argmax on ties.
8. XGBoost scale_pos_weight in multiclass — Binary-only parameter, silently ignored in multiclass. Fixed by computing sample_weight from compute_sample_weight('balanced') and passing it to fit().
9. No ROC curves for multiclass — Full plot_multiclass_roc() added: per-class OvR curves, macro interpolated AUC, and sklearn macro/weighted AUC.
10. No Focal Loss — Plain CrossEntropyLoss doesn't address inter-class imbalance (NONE is likely your dominant class). MulticlassFocalLoss(gamma=2.0) added as a drop-in with label_smoothing=0.1 support.
